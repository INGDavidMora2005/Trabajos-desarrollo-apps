# Plan for Flutter App: Taller 2 - Asincronía, Timer e Isolates

## Overview
Modify the existing Flutter project in the `taller2` directory to implement the required features for the workshop. Change the project name to `taller2_app` in `pubspec.yaml`. Implement demonstrations of asynchronous programming (Future/async/await), Timer for stopwatch, and Isolates for heavy computations.

## Phase 0: Git Setup
- Switch to `dev` branch
- Pull latest changes from `origin/dev`
- Create and checkout new feature branch `feature/taller_segundo_plano`

## Phase 1: Update Project Configuration
- Update `pubspec.yaml`: change name to `taller2_app`, update description to "Taller 2 - Asincronía, Timer e Isolates en Flutter", ensure dependencies include `cupertino_icons: ^1.0.2`, `flutter_lints: ^3.0.0` (adjust versions if needed)
- Run `flutter pub get` to update dependencies

## Phase 2: Create Project Structure
- Create the following directory structure under `lib/`:
  - `core/theme/`
  - `features/home/`
  - `features/async_demo/`
  - `features/timer/`
  - `features/isolate_demo/`
- Create empty files for all required components:
  - `lib/core/theme/app_theme.dart`
  - `lib/app.dart`
  - `lib/features/home/home_screen.dart`
  - `lib/features/async_demo/data_service.dart`
  - `lib/features/async_demo/async_screen.dart`
  - `lib/features/timer/timer_screen.dart`
  - `lib/features/timer/timer_controller.dart`
  - `lib/features/isolate_demo/isolate_screen.dart`
  - `lib/features/isolate_demo/heavy_task.dart`
  - `README.md` (root)

## Phase 3: Implement Theme and Global Configuration
- Implement `lib/core/theme/app_theme.dart` with Material 3 theme, ColorScheme.fromSeed using Colors.indigo, TextTheme with monospace font for timestamps, AppBarTheme, CardTheme with borderRadius 16
- Update `lib/main.dart` to import and run `TallerApp()`
- Implement `lib/app.dart` with `TallerApp` StatelessWidget returning MaterialApp with themes, system theme mode, HomeScreen as home, debug banner false

## Phase 4: Implement Home Screen
- Implement `lib/features/home/home_screen.dart` with Scaffold, AppBar "Taller 2 — Segundo Plano", body with Column/ListView of 3 Cards with InkWell for navigation to AsyncScreen, TimerScreen, IsolateScreen
- Each card has large icon, title, subtitle describing the feature
- Footer text "David Mora Duque — Desarrollo Apps Móviles"

## Phase 5: Implement Async Demo Feature
- Implement `lib/features/async_demo/data_service.dart` with class DataService containing methods:
  - `fetchData()`: Future<String> with 3s delay, success message
  - `fetchDataWithPossibleError()`: Similar but with random failure
  - `fetchMultipleData()`: Future.wait for multiple calls
  - All with console prints for progress
- Implement `lib/features/async_demo/async_screen.dart` with StatefulWidget, enum AsyncStatus, state for result, logs, buttons for each method, Card for status display (idle/loading/success/error), ListView for logs

## Phase 6: Implement Timer Feature
- Implement `lib/features/timer/timer_controller.dart` with TimerController class:
  - Fields: Timer?, elapsedMilliseconds, isRunning
  - Methods: start(resume), pause, reset, dispose
  - formattedTime getter for MM:SS:cc
- Implement `lib/features/timer/timer_screen.dart` with StatefulWidget, display time in large monospace font, color based on state, buttons for start/pause/resume/stop, laps list

## Phase 7: Implement Isolate Demo Feature
- Implement `lib/features/isolate_demo/heavy_task.dart` with IsolateMessage class and top-level function `heavyTaskIsolate` for CPU-bound tasks (sum or primes), sending progress and result via SendPort
- Implement `lib/features/isolate_demo/isolate_screen.dart` with StatefulWidget, enum IsolateStatus, state for progress, result, elapsed time, task selection (sum/primes), buttons to run in isolate or UI thread, display progress and result

## Phase 8: Create Documentation
- Implement `README.md` with full structure: description, project structure tree, when to use each tool, screen flows, how to run, dependencies, Git flow, etc.

## Phase 9: Testing and Verification
- Run `flutter analyze` to check for errors
- Run `flutter run` to test the app
- Ensure all features work: async operations show logs, timer runs/pauses, isolate computes without blocking UI

## Phase 10: Commits and Git
- After each major phase (3-8), commit with semantic messages like:
  - "feat: setup project structure and configuration"
  - "feat(async): implement data service and async screen"
  - "feat(timer): implement timer controller and screen"
  - "feat(isolate): implement isolate heavy task and screen"
  - "docs: add comprehensive README"
- Push the feature branch to remote

## Phase 11: Pull Request
- Create PR from `feature/taller_segundo_plano` to `dev` with checklist of implemented features, screenshots if possible, description of decisions

## Notes
- Ensure all code is in Spanish or English consistently (mix in prompt, but prefer English for code)
- Use try/catch for async error handling, mounted checks before setState
- Isolate function must be top-level
- UI responsive, no external packages beyond specified
- Follow GitFlow with feature branch</content>
